# 社区调研访谈资料整理

## 工作流程概览

```
原始访谈 → 逐条提取 → 维度编码 → 报告撰写 → 多社区对比 → Word 输出 → 完整性核验
```

## Phase 1: 原始资料提取

从用户提供的访谈转写文本（或截图）中逐条提取 Q&A 对。

**提取规范：**
- 保留受访者原话（口语化表述不修改）
- 标注每对 Q&A 的编号，便于后续核验
- 识别受访者基本信息（身份、居住时长、家庭特征）
- 如有多社区数据，标注每段访谈所属社区

## Phase 2: 八维度编码

将提取的 Q&A 归入以下八个分析维度（可根据实际数据增减）：

| 维度 | 关注点 |
|------|--------|
| 1. 制度筛选与进入方式 | 入住规则感知、身份门槛、信息对称性 |
| 2. 空间边界与日常行为 | 通勤模式、出行范围、时间分配 |
| 3. 资源依赖与生活配套 | 购物/餐饮/就医选择、商业配套满意度 |
| 4. 社交网络与邻里互动 | 社交圈构成、邻里互助、社区活动参与 |
| 5. 子女教育与成长环境 | 选址中的教育考量、儿童社交、课余安排 |
| 6. 心理归属与身份认同 | 家 vs 临时居所、身份认同、城市归属 |
| 7. 搬离成本与锁定机制 | 搬离顾虑、锁定因素、生活选择影响 |
| 8. 改进建议 | 配套诉求、管理建议、开放度偏好 |

**编码原则：**
- 一条 Q&A 可归入多个维度
- 标注受访者原话作为直接引语（使用""标注）
- 识别预期管理维度（如受访者提及"入住感受与预期对比"）

## Phase 3: 报告撰写

按以下结构输出单社区报告。详细模板见 [report-template.md](report-template.md)。

```
一、核心摘要（200字以内）
二、受访者画像（表格）
三、核心发现汇总（八维度逐项分析）
四、与研究框架的呼应（表格）
五、深度洞察（跨维度综合分析）
六、与研究设计关键词的关联
七、结论与建议方向
```

**撰写规范：**
- 每个维度先陈述事实（引用原话），再做分析（对比/推断）
- 区分"受访者表述"和"研究者分析"
- 识别遗漏维度（如受访者未涉及的话题也需注明）

## Phase 4: 多社区对比（仅多社区时）

当存在两个及以上社区数据时，新增对比分析章节：

```
一、N维度对比总表（表格，列：维度 | 社区A | 社区B | 差异性质）
二、对比分析要点（3-5段，聚焦核心差异与共性）
三、深度洞察（跨社区模式提炼）
```

**对比要点：**
- 差异性质用简短标签概括（如"高度一致""A更强""B痛点更突出"）
- 优先识别家庭结构分异效应（有子女 vs 无子女）
- 共性问题单独指出（如"邻里互助缺失"在多个社区同时出现）

## Phase 5: Word 文档输出

使用 docx 技能生成格式化文档。文档结构建议：

```
封面页
核心摘要
受访者画像表
Part A: 社区1 完整分析
Part B: 社区2 完整分析（如有）
Part C: 双社区对比分析（如有）
深度洞察
研究关键词关联
结论与建议
```

**格式要求：**
- 字体：Microsoft YaHei（中文兼容性最佳）
- 表格：深蓝色表头(#1F4E79) + 白色字，数据行交替底色
- 对比表的"差异性质"列用颜色标注（绿色=积极差异，橙色=痛点，蓝色=一致）
- 页眉页脚：项目名称 + 页码
- 使用 Heading1/2/3 层级，支持目录生成

## Phase 6: 完整性核验

报告完成后，执行逐项核验：

**核验清单：**
```
对每位受访者的每条 Q&A：
  □ 该信息是否出现在报告中？
  □ 如出现，是否完整保留关键细节？
  □ 受访者原话是否准确引用？

对研究框架：
  □ 每个理论维度是否有对应的访谈证据？
  □ 是否有遗漏的研究关键词？

对对比分析（多社区时）：
  □ 每个维度是否都做了对比？
  □ 正面和负面发现是否都有涵盖？
```

**核验输出格式：**
向用户报告遗漏清单，标注每条遗漏的严重程度（完全遗漏 / 部分遗漏），并主动提议修复。

## 常见问题处理

**访谈转写质量差：** 口语化文本中可能有错别字或断句不清，根据上下文推断原意，在报告中保留修正后的引语并加注。

**受访者信息不足：** 如居住时长、身份等关键信息缺失，在画像表中标注"未明确"而非猜测。

**维度不适用：** 某些维度可能不适用于特定受访者（如无子女受访者不涉及子女教育维度），标注"N/A"并在分析中说明。
